# Define MVP Contract

## Description
Choose the first Zendesk endpoints and output conventions so implementation starts from a fixed contract.

## Scope
Document auth inputs, initial read-only commands, output modes, and the initial repo layout for SKILL.md, references, and CLI code.

## Acceptance Criteria
MVP covers a small read-only Zendesk slice, auth inputs are explicit, command names are chosen, and the repo scaffold plan is concrete.
